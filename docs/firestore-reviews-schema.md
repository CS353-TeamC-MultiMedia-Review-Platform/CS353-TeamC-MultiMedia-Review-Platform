# Firestore Reviews Collection Schema

## Collection Name: `reviews`

### Document Structure

Each review document in the `reviews` collection follows this schema:

```typescript
interface Review {
  // Required Fields
  id: string; // Auto-generated document ID (Firebase)
  userId: string; // UID of the user who created the review (from Firebase Auth)
  userName: string; // Display name of the reviewer
  rating: number; // Rating from 1 to 5 (integer)
  reviewText: string; // Main review content (min 10 chars, max 2000 chars)
  mediaTitle: string; // Title of the media being reviewed
  mediaType: "movie" | "music" | "book"; // Type of media
  createdAt: Timestamp; // Firebase Timestamp of review creation

  // Optional Fields
  mediaId?: string; // External ID for the media (e.g., IMDb ID, ISBN)
  updatedAt?: Timestamp; // Firebase Timestamp of last update
  tags?: string[]; // Optional tags for categorization (e.g., ["action", "sci-fi"])
  helpful?: number; // Helpful votes counter (default: 0)
  unhelpful?: number; // Unhelpful votes counter (default: 0)
}
```

### Field Validation Rules

| Field        | Type      | Required | Min/Max    | Rules                              |
| ------------ | --------- | -------- | ---------- | ---------------------------------- |
| `id`         | String    | Yes      | -          | Auto-generated by Firestore        |
| `userId`     | String    | Yes      | -          | Must be valid Firebase Auth UID    |
| `userName`   | String    | Yes      | 1-100      | Non-empty, trimmed string          |
| `rating`     | Number    | Yes      | 1-5        | Integer only                       |
| `reviewText` | String    | Yes      | 10-2000    | Non-empty, must have min length    |
| `mediaTitle` | String    | Yes      | 1-255      | Non-empty, trimmed string          |
| `mediaType`  | String    | Yes      | -          | One of: 'movie', 'music', 'book'   |
| `createdAt`  | Timestamp | Yes      | -          | Server-set timestamp               |
| `mediaId`    | String    | No       | -          | External identifier                |
| `updatedAt`  | Timestamp | No       | -          | Server-set timestamp on update     |
| `tags`       | Array     | No       | 0-10 items | String array, max 50 chars per tag |
| `helpful`    | Number    | No       | ≥ 0        | Non-negative integer               |
| `unhelpful`  | Number    | No       | ≥ 0        | Non-negative integer               |

### Collection Indexes

**Recommended indexes for optimal query performance:**

1. **Primary Index**
   - Collection: `reviews`
   - Fields: `createdAt` (Descending)
   - Purpose: Fetch latest reviews

2. **User Reviews Index**
   - Collection: `reviews`
   - Fields: `userId` (Ascending), `createdAt` (Descending)
   - Purpose: Get all reviews by a specific user

3. **Media Reviews Index**
   - Collection: `reviews`
   - Fields: `mediaTitle` (Ascending), `createdAt` (Descending)
   - Purpose: Get all reviews for a specific media

4. **Type & Rating Index**
   - Collection: `reviews`
   - Fields: `mediaType` (Ascending), `rating` (Descending)
   - Purpose: Browse reviews by media type and rating

### Security Rules

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /reviews/{document=**} {
      // Allow anyone to read reviews
      allow read: if true;

      // Allow authenticated users to create reviews
      allow create: if request.auth != null
        && request.resource.data.userId == request.auth.uid
        && request.resource.data.keys().hasAll(['userId', 'userName', 'rating', 'reviewText', 'mediaTitle', 'mediaType', 'createdAt']);

      // Allow users to update/delete only their own reviews
      allow update, delete: if request.auth != null
        && resource.data.userId == request.auth.uid;
    }
  }
}
```

### Firestore Paths

- **Single Review**: `/reviews/{reviewId}`
- **User's Reviews**: Query where `userId == "{uid}"` and order by `createdAt`
- **Media Reviews**: Query where `mediaTitle == "{title}"` and order by `rating`
- **Recent Reviews**: Query all documents ordered by `createdAt` (descending)

### Notes

1. **Timestamps**: Use Firebase Admin SDK's `admin.firestore.Timestamp.now()` for server-side timestamp accuracy
2. **User Validation**: Always validate that `userId` matches the authenticated user making the request
3. **Data Consistency**: Ensure review data is validated before saving to Firestore
4. **Deletion Handling**: Consider marking reviews as deleted instead of hard-deleting for data retention
5. **Rating Distribution**: Consider maintaining a summary collection for quick access to average ratings by media
